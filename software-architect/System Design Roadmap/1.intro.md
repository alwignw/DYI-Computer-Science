---

## 🧠 Inti Besarnya Dulu

Kalimat ini:

> *System design is the process of defining the elements of a system, as well as their interactions...*

👉 Artinya:
**System design = proses merancang sebuah sistem dari bagian kecil sampai jadi satu kesatuan yang bekerja**

---

## 🔧 Breakdown Supaya Lebih Kebayang

### 1. **"defining the elements of a system"**

👉 Kita menentukan **komponen sistem**

Contoh:
Kalau bikin aplikasi e-commerce:

* Frontend (web/mobile)
* Backend (API)
* Database
* Payment service
* Notification service

➡️ Ini disebut **building blocks**

---

### 2. **"their interactions and relationships"**

👉 Bukan cuma bikin komponen, tapi juga:
**bagaimana mereka saling komunikasi**

Contoh:

* Frontend → API → Database
* Backend → Payment Gateway
* Backend → Kirim email

➡️ Ini disebut **flow / komunikasi antar sistem**

---

### 3. **"to satisfy requirements"**

👉 Semua desain harus memenuhi kebutuhan

Contoh requirement:

* Bisa handle 1 juta user
* Response cepat (<200ms)
* Aman (security)

➡️ Jadi design itu bukan asal bagus, tapi harus:
**solve problem + memenuhi kebutuhan**

---

## 🧩 Kalimat Kedua (lebih dalam)

> *breaking it down into smaller components...*

👉 Ini mindset penting:
**Masalah besar → dipecah jadi kecil**

Contoh:
"Bangun sistem chat seperti WhatsApp"

Dipecah jadi:

* Auth system
* Chat service
* Message queue
* Database
* Notification

➡️ Ini skill utama System Design:
**decomposition**

---

## 🔄 Iterative Process

> *design, testing, refinement*

👉 Artinya:
System design itu **tidak sekali jadi**

Flow-nya:

1. Design awal
2. Test / simulasi
3. Ketemu masalah
4. Perbaiki

➡️ Ulang terus sampai optimal

---

## 🏗️ Dalam Software Engineering

> *focus on high-level design*

👉 Ada 2 level:

### 1. High-Level Design (HLD)

* Arsitektur (Monolith / Microservices)
* Komponen utama
* Data flow

### 2. Low-Level Design (LLD)

* Detail coding
* Class / function
* Database schema

➡️ System design fokus ke:
👉 **High-Level dulu**

---

## 💼 Kenapa penting buat interview?

Perusahaan suka nanya:

👉 "Design system seperti Instagram"
👉 "Design URL shortener (bit.ly)"

Yang dinilai:

* Cara berpikir
* Cara memecah masalah
* Trade-off (misalnya: cepat vs murah)

---

## ⚖️ Trade-off (ini kunci banget)

Contoh:

* Mau cepat → pakai cache (Redis)
* Tapi jadi lebih kompleks

👉 Jadi tidak ada solusi sempurna
➡️ Selalu ada kompromi

---

## 🔥 Ringkasan Super Singkat

System design itu:

1. Memahami problem
2. Memecah jadi komponen
3. Menentukan hubungan antar komponen
4. Mendesain agar memenuhi requirement
5. Iterasi sampai optimal

---

## 🎯 Analoginya biar makin nempel

Bayangin bangun rumah:

* Komponen → kamar, dapur, listrik
* Relasi → kabel nyambung ke lampu
* Requirement → tahan gempa, muat 5 orang

👉 Itu sama persis dengan system design, tapi versi software
