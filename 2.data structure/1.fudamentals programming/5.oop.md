**Object-Oriented Programming (OOP)** itu adalah **cara berpikir dan menulis kode dengan konsep “objek”**.
Objek di sini kayak **benda nyata** — punya **ciri-ciri (data/properti)** dan **perilaku (aksi/fungsi)**.

---

### 🔹 Inti dari OOP

Dalam OOP, kamu bikin **class (kelas)** sebagai *cetakan* atau *template*,
lalu dari class itu kamu bisa buat **object (objek)** sebagai *contohnya*.

Contoh gampang:

> Class: “Mobil”
> Object: mobil1, mobil2, mobilBos 😄

**Class** mendefinisikan apa aja yang dimiliki semua mobil (misalnya warna, merk, kecepatan)
dan apa yang bisa dilakukan (misalnya nyalakanMesin(), jalan(), rem()).

---

### 🔹 Istilah penting di OOP

1. **Class**
   Template yang mendefinisikan struktur dan perilaku objek.
   Misalnya:

   ```python
   class Mobil:
       def __init__(self, warna, merk):
           self.warna = warna
           self.merk = merk

       def nyalakan(self):
           print("Mesin dinyalakan")
   ```

2. **Object**
   Wujud nyata dari class.
   Contoh:

   ```python
   m1 = Mobil("Merah", "Toyota")
   m1.nyalakan()
   ```

3. **Encapsulation (Enkapsulasi)**
   Menyembunyikan detail internal agar **data tidak bisa diubah sembarangan**.
   Kayak kamu menaruh mesin dalam kap mobil — orang luar bisa pakai mobilnya, tapi gak perlu tahu cara kerja mesin di dalamnya.

4. **Inheritance (Pewarisan)**
   Class baru bisa **mewarisi** sifat dan perilaku dari class lain.
   Misalnya class `MobilBalap` bisa mewarisi dari `Mobil`, tapi menambah fitur `turbo()`.

5. **Polymorphism (Polimorfisme)**
   Artinya satu perintah bisa punya **bentuk berbeda** tergantung objeknya.
   Misalnya, fungsi `jalan()` bisa punya perilaku berbeda di `Mobil` dan `Motor`.

---

### 🧠 Kesimpulan:

> **OOP** = cara menulis program dengan mengelompokkan data dan perilaku ke dalam *objek*, biar kode lebih terstruktur, mudah diatur, dan mirip dengan dunia nyata.
